# Agent Notes

## Commands

- Root scripts are only `pnpm dev` and `pnpm build`; both run recursively across workspaces.
- Focused app commands:
  - `pnpm --filter regarde.inspector dev` starts the inspector via `portless regarde.inspector vite` and also starts `agentation-mcp server`.
  - `pnpm --filter regarde.inspector build` runs `vite build && tsc`.
  - `pnpm --filter regarde.inspector typecheck` runs `tsc --noEmit`.
  - `pnpm --filter my-jazz-app dev` runs the sample Jazz app with the local inspector plugin.
- Focused package commands:
  - `pnpm --filter @regarde/ui build|dev|typecheck|lint|format`.
  - `pnpm --filter @regarde/jazz-dev-tools build|dev|typecheck`.
- There are no test scripts or test configs in the manifests; use typecheck/build/lint where available.

## Workspace map

- `apps/web-inspector` is the standalone React inspector. Entry is `src/app.tsx`; routes are file-based under `src/routes`.
- `apps/web-inspector/src/routeTree.gen.ts` is generated by TanStack Router; do not hand-edit it.
- `packages/ui` is the private UI library consumed as `@regarde/ui`.
- `packages/jazz-dev-tools` exports the Vite `jazzInspectorPlugin` and inspector link helpers.
- `apps/my-jazz-app` is a sample app using `jazzInspectorPlugin()` from `@regarde/jazz-dev-tools`.

## Architecture constraints

- The inspector is schema-driven and generic. Do not add inspected-app generated query builders or table-specific UI; use stored schema metadata, generic query construction, `useAll(...)`, and `useDb()` patterns instead.
- Keep inspector-specific data logic in `apps/web-inspector`; keep reusable presentational components in `packages/ui`.
- Connection data includes `serverUrl`, `appId`, `adminSecret`, branch, and schema hash. Treat `adminSecret` as sensitive even though local dev links pass it in URL hash params.

## UI package rules

- `packages/ui/src/components/ui` contains generated shadcn/Base UI components. Do not edit these directly; regenerate them.
- Put branded reusable components in `packages/ui/src/components/atoms` using camelCase filenames.
- Atom wrappers should import shadcn components with a `Shad` alias, e.g. `Input as ShadInput`.
- Atom props should derive from `React.ComponentProps<typeof ShadComponent>`; React 19 refs are already included in those props.
- Use `cn()` for class merging in atoms; reserve `twMerge()` for CVA variant definitions.

## TypeScript conventions

- Always use explicit boolean comparisons: `=== true` and `=== false`. Do not use implicit truthiness such as `if (value)`, `if (!value)`, or `!!value`.
- Preserve strict settings: `noUnusedLocals`, `noUnusedParameters`, `noUncheckedSideEffectImports`, and `erasableSyntaxOnly` where configured.
- Prefix native Node imports with `node:`; this is already used in `packages/jazz-dev-tools`.
