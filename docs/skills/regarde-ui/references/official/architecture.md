# Architecture

Three-layer system for building consistent, themeable UI components.

## The Layers

```
┌─────────────────────────────────────┐
│  /atoms/                            │  ← Branded components (your code)
│  - Domain-specific features         │
│  - Semantic Tailwind classes        │
│  - Regarde styling                  │
├─────────────────────────────────────┤
│  /ui/                        │  ← Raw shadcn (pristine)
│  - BaseUI primitives with styling   │
│  - Generated by shadcn CLI          │
│  - DO NOT EDIT                      │
├─────────────────────────────────────┤
│  /base-ui/                          │  ← Base UI utilities
│  - mergeProps, useRender            │
│  - Re-exports from @base-ui/react   │
│  - Thin wrapper layer               │
└─────────────────────────────────────┘
```

## Layer Details

### /base-ui/

Thin re-exports of Base UI utilities:

| File             | Exports      | Usage                           |
| ---------------- | ------------ | ------------------------------- |
| `merge-props.ts` | `mergeProps` | Merge internal and user props   |
| `use-render.ts`  | `useRender`  | Polymorphic component rendering |

**Import from `/base-ui/` for utilities, from `@base-ui/react/*` for primitives.**

### /ui/

Raw shadcn components generated via CLI. These wrap BaseUI primitives with styling.

**Critical rules:**

- Never edit these files directly
- Regenerate with `shadcn add --overwrite`
- Import with `Shad` prefix in atoms
- Do not add exports (breaks on regeneration)

### /atoms/

Your branded components. This is where you:

- Add domain-specific features (slots, loading states)
- Apply semantic styling
- Create reusable patterns

**Editable**: Yes, this is Regarde code.

## Data Flow

```
App imports:     @regarde/ui/components/atoms/button
                      ↓
Atom wraps:      ShadButton (from /ui/)
                      ↓
Shadcn wraps:    BaseButton (from @base-ui/react)
                      ↓
Browser renders: Styled component
```

## Theme System

**No CSS files in UI package** — only Tailwind classes.

Apps control branding via CSS variables:

```css
/* apps/dashboard/src/index.css */
@theme {
  --color-input: var(--input);
  --color-border: var(--border);
  --color-foreground: var(--foreground);
}

:root {
  --input: oklch(1 0 0);
  --border: oklch(0.922 0 0);
  --foreground: oklch(0.145 0 0);
}
```

Atoms use semantic classes:

```typescript
className={cn(
  "bg-input border-border text-foreground",
  className
)}
```

## Common Semantic Tokens

| Token                     | Purpose                     |
| ------------------------- | --------------------------- |
| `bg-input`                | Form input backgrounds      |
| `border-border`           | Default borders             |
| `text-foreground`         | Primary text color          |
| `text-muted-foreground`   | Secondary text color        |
| `bg-background`           | Page/component backgrounds  |
| `bg-primary`              | Primary action backgrounds  |
| `text-primary-foreground` | Text on primary backgrounds |
| `focus-visible:ring-ring` | Focus ring color            |

## Class Merging Strategy

### CVA Components

For components using CVA (Button, Badge), pass `className` to CVA, not `twMerge` directly:

```typescript
// CORRECT - CVA handles className internally and always returns a string
const baseClasses = twMerge(buttonVariants({ variant, size, className }));

// WRONG - twMerge can't handle function className from Base UI
const baseClasses = twMerge(buttonVariants({ variant, size }), className);
```

### Non-CVA Components

For simple components without CVA, use `cn()` for conditional classes:

```typescript
className={cn(
  "bg-input border-border",
  preSlot && "pl-9",
  className
)}
```

### Rule of Thumb

| Tool        | Use When                                  |
| ----------- | ----------------------------------------- |
| `twMerge()` | CVA variants only (pass className to CVA) |
| `cn()`      | Conditional logic (`&&`, ternaries)       |

## Prerequisites

1. Verify Base UI project by checking `components.json`:
   - `style` field should start with `"base-*"` (e.g., `"base-vega"`)
2. Ensure `@base-ui/react` is installed
